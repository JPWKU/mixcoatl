set_role(obj, *args, **kwargs)[source]¶

Updates the role applied to the group specified by group_id. account_id is technically optional, but we’re making it required here because making it optional may result in user problems.

It’s complicated.

If you don’t specify an account then it’ll find an account associated with the billing credentials for the given user.
