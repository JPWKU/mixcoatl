snapshot(**kwargs)[source]¶

Make a snapshot from a volume

This is not a part of the enStratus API for Volume resources. It is a convenience wrapper around the Snapshot API from the perspective of an individual Volume

Warning

Snapshot creation is an asynchronous task. Specifying a callback will cause a blocking operation while the snapshot completes When using the callback, execution could block for a VERY long time.

Parameters:
  • name (str.) – The name to assign the Snapshot. Default: snap-<volume_id>-timestamp
  • description (str.) – Description of the snapshot. Default: mixcoatl snapshot
  • budget (int.) – Budget to assign the snapshot. Default: budget of the current Volume
  • callback (func.) – Optional callback to return the results.
Returns:

Snapshot
